# Release anfordern {#request-a-release}

## Voraussetzungen  {#prerequisites}

* Sie haben die Rolle **Release-Manager** - siehe <!-- broken link[User roles](../teams/user-roles.md) -->
* Ihr Programm wurde integriert - siehe [Onboarding Ihres Programms](../applications/onboard-your-application.md)

>[!TIP]
>
>Bevor Sie eine Version anfordern, überprüfen Sie <!--[Releases and cross-team feature groups](releases-and-cross-team-feature-groups.md)-->. Eine Team-übergreifende Funktionsgruppe kann Ihren Anforderungen mit weniger Aufwand gerecht werden - sie stellt sich selbst bereit und unterstützt eine umfassendere Zielgruppen-Zielgruppenbestimmung.

## Senden einer Support-Anfrage {#submit}

Die Erstellung von Versionen erfolgt nicht im Self-Service. Kontaktieren Sie den Flags-Support, um eine neue Version anzufordern. Geben Sie die folgenden Informationen an:

| Feld | Beschreibung |
|---|---|
| **Name** | Ein eindeutiger Name für die Version. Verwenden Sie keine Leerzeichen im Release-Namen. |
| **Team** | Das Team, dem die Version gehört. |
| **Umgebung** | Die Umgebung, in der die Version erstellt werden soll (Staging- oder Produktionsumgebung). |
| **Ziel** | Eine kurze Beschreibung dessen, was diese Version herausbringen wird und warum. |
| **Anwendungen** | Die Anwendung(en), die in die Version aufgenommen werden sollen. |
| **Dauer** | Wie lange die Version benötigt wird, um aktiv zu bleiben. Es wird erwartet, dass die Versionen innerhalb von drei Monaten geschlossen oder als Baseline festgelegt werden. Wenn Sie länger benötigen, geben Sie eine geschäftliche Begründung an. |

>[!NOTE]
>
>Die Anzahl der aktiven Releases zu einem bestimmten Zeitpunkt ist begrenzt. Planen Sie die Veröffentlichung innerhalb von drei Monaten ein oder schließen Sie sie, um Kapazitäten für andere Teams freizugeben.

## Nachdem die Version erstellt wurde {#after}

Nachdem Sie die Bestätigung erhalten haben, dass die Version erstellt wurde, melden Sie sich bei der Konsole an und schließen Sie die Konfiguration der Version ab. Siehe [End-to-End-Workflow von Version](release-workflow-end-to-end.md) für die vollständige Abfolge der Schritte.
