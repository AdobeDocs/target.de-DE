## Welche Funktion hat sie? {#section_4ECAACC68723402EB3649033190E1BBC}

| Aktivitätstyp | Details |
|--- |--- |
| Manuell [!UICONTROL A/B-Test]<P>![icon](/help/main/c-activities/assets/icon_ab.png) | Vergleicht zwei oder mehr Erlebnisse, um festzustellen, durch welches Erlebnis die Konversionen über einen zuvor festgelegten Testzeitraum am besten verbessert werden.<P>Weitere Informationen finden Sie unter [A/B-Test](/help/main/c-activities/t-test-ab/test-ab.md). |
| [!UICONTROL Automatische Zuordnung]<P>![Symbol Automatische Zuordnung](/help/main/c-activities/assets/icon_auto_allocate.png) | Identifiziert einen Gewinner unter zwei oder mehr Erlebnissen und leitet dann Traffic zum erfolgreichsten Erlebnis um, wodurch die Konversion im Zuge der Testausführung und des Lernens erhöht wird.<P>Weitere Informationen finden Sie unter [Automatische Zuordnung](/help/main/c-activities/automated-traffic-allocation/automated-traffic-allocation.md). |
| [!UICONTROL Automatisches Targeting]<P>![Symbol AT](/help/main/c-activities/assets/icon_auto_target.png) | Verwendet das erweiterte maschinelle Lernen zum Personalisieren von Inhalt und Fördern von Konversionen, indem mehrere vom Vermarkter definierte Erlebnisse mit hoher Leistung identifiziert werden und das passendste Erlebnis für Benutzer bereitgestellt wird. Dies basiert auf den individuellen Kundenprofilen und dem Verhalten ähnlicher vorheriger Besucher.<P>Weitere Informationen finden Sie unter [Automatisches Targeting für personalisierte Erfahrungen](/help/main/c-activities/auto-target/auto-target-to-optimize.md). |
| [!UICONTROL Automated Personalization] (AP)<P>![Symbol AP](/help/main/c-activities/assets/icon_ap.png) | Verwendet das erweiterte maschinelle Lernen zum Personalisieren von Inhalt und Fördern von Konversionen, indem spezielle Angebote oder Nachrichten kombiniert und anschließend basierend auf den individuellen Kundenprofilen verschiedene Angebotsvarianzen zugeordnet werden.<P>Weitere Informationen finden Sie unter [Automated Personalization](/help/main/c-activities/t-automated-personalization/automated-personalization.md). |
| [!UICONTROL Multivariate Tests] (MVT)<P>![Symbol MVT](/help/main/c-activities/assets/icon_mvt.png) | Vergleicht Angebotskombinationen zwischen Elementen auf einer Seite, um festzustellen, welche Angebotskombination für eine bestimmte Zielgruppe die beste Leistung erzielt. Identifiziert zudem das Element der Seite, durch das für Konversionen über einen zuvor definierten Zeitraum die besten Verbesserungen erzielt werden.<P>Weitere Informationen finden Sie unter [Multivarianz-Test](/help/main/c-activities/c-multivariate-testing/multivariate-testing.md). |
| Erlebnis-Targeting (XT)<P>![Symbol XT](/help/main/c-activities/assets/icon_xt.png) | Stellt Inhalte für eine spezielle Zielgruppe basierend auf einem Satz aus vermarkterdefinierten Regeln und Kriterien bereit.<P>Weitere Informationen finden Sie unter [Erlebnis-Targeting](/help/main/c-activities/t-experience-target/experience-target.md). |

## Warum verwenden Sie diese Art von Aktivität? {#section_46A70DD7CE3448749E635DDF5EAFC131}

| Aktivitätstyp | Grund |
|--- |--- |
| Manuell [!UICONTROL A/B-Test] | Ein stark gesteuertes Experiment mit Traffic-Messungen, die nach Prozentsätzen statt nach einer Regel aufgeteilt sind. Es ermöglicht Ihnen das Analysieren der Testdaten, das Sammeln von Insights zu Ihrer Zielgruppe und das Bestimmen des Erlebnisses mit der besten Leistung. |
| [!UICONTROL Automatische Zuordnung] | Eine Möglichkeit, ein erfolgreichstes Erlebnis zu identifizieren und die Traffic-Zuordnung so anzupassen, dass das erfolgreichste Erlebnis so schnell wie möglich für Besucher bereitgestellt wird. Dies ermöglicht eine schnellere und höhere Konversionswahrscheinlichkeit. |
| [!UICONTROL Automatisches Targeting] | Eine Methode zum Identifizieren der Gewinner unter mehreren Erlebnissen und zum anschließenden Bereitstellen des am besten geeigneten Erlebnisses für spezifische Besucher. Das Targeting wird im Laufe der Zeit im Zuge der sich verändernden Besucherinteressen angepasst, weil der Algorithmus die Neigung eines Besuchers zur Konversion für ein bestimmtes Erlebnis in einer bestimmten Zeit vorhersagt. |
| [!UICONTROL Automated Personalization] (AP) | Eine Methode zum Personalisieren eines Angebotssatzes (erstellt oder vordefiniert, in Elementen auf einer einzelnen oder auf mehreren Seiten) und Bereitstellen von Angebotskombinationen, mit denen Besucher am besten angelockt werden. |
| [!UICONTROL Multivariate Testing] (MVT) | Eine Möglichkeit, mehrere Angebote in mehreren Elementen anzuzeigen und dann die resultierenden eindeutigen Erlebnisse gleichzeitig mit einem bestimmten Ziel zu testen, was dazu beiträgt, zu ermitteln, welche Elementvariante am erfolgreichsten ist. Eine MVT-Aktivität kann auch zeigen, welche Elemente die größten positiven oder negativen Auswirkungen auf die Interaktion eines Besuchers haben. |
| [!UICONTROL Experience Targeting] (XT) | Eine Möglichkeit, bestimmte Inhalte basierend auf definierten Zuordnungsregeln auf eine bestimmte Zielgruppe auszurichten. |

## Welche Art von Marketingexperten sollte eine Aktivität verwenden? {#section_A843D663D3E543FFB1A594266B560395}

| Aktivitätstyp | Vermarkter |
|--- |--- |
| Manuell [!UICONTROL A/B-Test] | Kennt sich mit Statistiken aus.<P>Hat die Zeit, das Ende des Testzeitraums abzuwarten, um die Ergebnisse zu analysieren. |
| [!UICONTROL Automatische Zuordnung] | Hat eine kurzes Zeitfenster.<P>Muss das beste Erlebnis ermitteln und schnell bereitstellen.<P>Möchte in der Lage sein, während der Testausführung einen Blick auf die Ergebnisse zu werfen. |
| [!UICONTROL Automatisches Targeting] | Hat verschiedene qualifizierte Erlebnisse.<P>Möchte speziellen Besuchern zur optimalen Zeit Erlebnisse basierend auf ihren dynamischen und sich ändernden Profilen zuordnen. |
| [!UICONTROL Automated Personalization] (AP) | Hat ein oder mehrere Angebote.<P>Möchte Angebotskombinationen erstellen, die für bestimmte Besucher über verschiedene eindeutige Profile und Verhaltensweisen hinweg optimale personalisierte Erlebnisse liefern. |
| [!UICONTROL Multivariate Testing] (MVT) | Kennt sich mit Statistiken aus.<P>Hat ein oder mehrere Angebote.<P>Möchte Konversionstrends bezüglich der Interaktionen mit Seitenelementen analysieren. |
| [!UICONTROL Experience Targeting] (XT) | Muss ein bestimmtes Erlebnis oder einen bestimmten Inhalt für eine bestimmte Zielgruppe bereitstellen. |

## Statistische Details {#section_22CF2D07DB054505AB5EC702B99A5BB0}

| Aktivitätstyp | Details |
|--- |--- |
| Manuell [!UICONTROL A/B-Test] | Der Test vergleicht jedes Challenger-Erlebnis mit einem Kontrollerlebnis und ordnet dann die Leistung aller Erlebnisse an, wobei sowohl ein erfolgreiches als auch ein verlorenes Erlebnis im Vergleich zum Kontrollerlebnis identifiziert werden. |
| [!UICONTROL Automatische Zuordnung] | Der Test erzeugt direkt eine statistische Garantie für einen echten Gewinner und leitet dann mithilfe des erfolgreichsten Erlebnisses mehr Traffic zu den Zielgruppen mit höherer Konversionswahrscheinlichkeit. |
| [!UICONTROL Automatisches Targeting] | Der Optimierungsmechanismus identifiziert die relevante Zielgruppe für jedes Erlebnis, indem er im Laufe der Zeit Steigerungen und Rückgänge anzeigt und bevor er bestimmt, welches Erlebnis für welchen Besucher bereitgestellt werden soll. Der Optimierungsmechanismus wird durch Konversionen, Segmente, Parameter und Profilskripte informiert. Danach wählt der Mechanismus automatisch, welchen Algorithmus verwendet werden soll, um eine höhere Steigerung und Konversionsrate zu generieren. |
| [!UICONTROL Automated Personalization] (AP) | Der Optimierungsmechanismus passt fortlaufend an, welche Erlebnisse für welche Besucher bereitgestellt werden. Dies basiert auf dem Verhalten neuer Besucher und dem bisherigen Verhalten ähnlicher Besucher, wobei die Leistung eines Angebots anhand gleichzeitiger Kontrollgruppen gemessen wird. |
| [!UICONTROL Multivariate Testing] (MVT) | Der Test hilft bei der Bestimmung des relativen Einflusses, den spezielle Elemente auf die Konversion haben. |
| [!UICONTROL Experience Targeting] (XT) | Die Methode definiert Regeln für das Targeting eines speziellen Erlebnisses oder eines speziellen Inhalts für eine spezifische Zielgruppe. Kunden können Aktualisierungen auf Erlebnisebene vornehmen. |

## Vorteile und Überlegungen {#section_56C46ABEF7B945DDA0C1E6D714377123}

| Aktivitätstyp | Vorteile | Zu beachten |
|--- |--- |--- |
| Manuell [!UICONTROL A/B-Test] | Mit A/B-Ruhezeiten erhalten Sie ein umfassendes Verständnis davon, wie die einzelnen Erlebnisse funktionieren, und können nicht nur ermitteln, welche Erlebnisse die besten Ergebnisse erzielen. | In einer [!UICONTROL A/B-Test]Wenn Sie sich die Testergebnisse ansehen, bevor die Stichprobengröße erreicht ist, riskieren Sie, sich auf ungenaue Ergebnisse zu verlassen (Sie können nicht früher &quot;gucken&quot;!).<P>Unlike [!UICONTROL Automatische Zuordnung]festgelegt ist, bleibt die Traffic-Verteilung bei einem A/B-Test fest, auch wenn Sie erkennen, dass einige Erlebnisse eine bessere Leistung erzielen als andere.<P>Informationen zu Best Practices für [!UICONTROL A/B-Test] Aktivitäten, siehe [Dauer der Durchführung eines A/B-Tests](/help/main/c-activities/t-test-ab/sample-size-determination.md) und [Zehn häufige Fehler bei A/B-Tests und wie diese vermieden werden](/help/main/c-activities/t-test-ab/common-ab-testing-pitfalls.md). |
| [!UICONTROL Automatische Zuordnung] | [!UICONTROL Durch das automatische Zuweisen werden die Kosten eines typischen A/B-Tests gesenkt, weil er eine höhere Gesamtkonversionsrate bietet als ein manueller A/B-Test. ] Die Konversionsrate ist höher, weil [!UICONTROL Automatische Zuordnung] sendet mehr Traffic an das leistungsstärkste Erlebnis, d. h. Sie können den Vorteil dieses erfolgreichsten Erlebnisses vor dem Ende des Testzeitraums realisieren (Sie können einen Blick darauf werfen!). | [!UICONTROL Beim automatischen Zuweisen wird zwar der Gewinner identifiziert, unter den Verlierern erfolgt jedoch keine Differenzierung. ] Wenn Sie wissen müssen, wie jedes Erlebnis funktioniert hat, ist ein A/B-Test vorzuziehen.<P>Die [!UICONTROL Automatische Zuordnung] -Funktion kann nur mit einer erweiterten Metrikeinstellung verwendet werden, nämlich &quot;Anzahl erhöhen und Benutzer in Aktivität belassen&quot;. Wenn Sie keine wiederholten Konversionen zählen möchten, sollten Sie stattdessen A/B-Tests verwenden. |
| [!UICONTROL Automatisches Targeting] | Mit [!UICONTROL Automatisches Targeting], wird maschinelles Lernen auf jede Art von Erlebnis angewendet, einschließlich mehrseitiger Erlebnisse. Ein [!UICONTROL Automatisches Targeting] -Aktivität können Sie außerdem den Wert von [!UICONTROL Automated Personalization] bei Verwendung des bekannten A/B-Test-Workflows. | Mit [!UICONTROL Automatisches Targeting]Wenn Sie den Inhalt Ihrer Angebote häufig oder häufig ändern möchten, benötigt der Algorithmus genügend Zeit nach jeder Änderung, um das Erlernte zu nutzen und den richtigen Besuchern den Inhalt bereitzustellen. |
| [!UICONTROL Automated Personalization] (AP) | Mit [!UICONTROL Automated Personalization]können Sie alle Angebote an einem Ort erfassen und der Algorithmus bestimmt die beste Kombination von Angeboten. Die Angabe oder Erstellung individueller Erlebnisse ist nicht erforderlich. [!UICONTROL Automated Personalization] verwendet dieselben maschinellen Lernalgorithmen wie [!UICONTROL Automatisches Targeting]. | Wenn Sie mehrere Angebote kombinieren, kommt es zu einer kombinatorischen Explosion, für die wiederum viel Traffic erforderlich ist. Die [!UICONTROL Automated Personalization] -Algorithmus berücksichtigt viele Faktoren, sodass der größte Traffic erforderlich ist.<P>[!UICONTROL Automated Personalization] kann keine Berichte in [Analytics for Target](/help/main/c-integrating-target-with-mac/a4t/a4t.md) (A4T). |
| [!UICONTROL Multivariate Testing] (MVT) | Mit [!UICONTROL Multivariate Testing], können Sie mehrere Elemente gleichzeitig testen. | A [!UICONTROL Multivarianz-Test] ist zeitaufwendig, und aufgrund der zahlreichen Variablen im Spiel generiert es nicht unbedingt ein Gewinnererlebnis mit Konfidenz.<P>Den zum Abschließen des Tests benötigten Traffic zu erreichen ist oftmals eine große Herausforderung. Alle [!UICONTROL Multivarianz-Test] -Experimente sind vollfaktoriell. Zu viele sich gleichzeitig ändernde Elemente können schnell zu vielen möglichen Kombinationen führen, die getestet werden müssen.<P>Selbst bei einer Website mit ziemlich hohem Traffic-Aufkommen kann es schwierig werden, einen Test mit mehr als 25 Kombinationen innerhalb einer vernünftigen Zeit abzuschließen. |
| [!UICONTROL Experience Targeting] (XT) | Mit [!UICONTROL Erlebnis-Targeting]können Sie schnell auf Einblicke reagieren, die aus beliebigen Aktivitätsergebnissen abgeleitet wurden.<P>Wenn Sie beispielsweise einen A/B-Test durchgeführt haben, bei dem der Challenger die Kontrolle nicht übertroffen hat, die Ergebnisse jedoch darauf hindeuten, dass ein bestimmtes Besuchersegment mit dem Challenger viermal so viel konvertierte wie mit der Kontrolle, können Sie [!UICONTROL Erlebnis-Targeting] , um das Challenger-Erlebnis an dieses bestimmte Segment weiterzuleiten. | [!UICONTROL Beim Erlebnis-Targeting können Sie die Prozentaufteilung eines Erlebnisses über mehrere Zielgruppen nicht kontrollieren.] |
